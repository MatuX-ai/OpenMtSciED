## 🔍 根本原因分析

### 问题 1: 健康检查端点返回 503

**根本原因**:
```python
# 错误代码
db.execute("SELECT 1")  # SQLAlchemy 2.0+ 需要 text() 包装

# 正确代码
from sqlalchemy import text
db.execute(text("SELECT 1"))
```

**修复方案**: ✅ 已在 `backend/openmtscied/main.py` 中修复

---

### 问题 2: 用户登录失败 (401)

**可能原因**:
1. Mock 用户数据未正确初始化
2. 测试使用的密码与存储的哈希不匹配
3. bcrypt 验证逻辑有问题

### 问题 3: 性能问题

**观察**:
- API 响应时间超过 2000ms（目标 < 200ms）
- 并发测试全部失败

**可能原因**:
1. **速率限制过于严格**: slowapi 限制为 5次/分钟
2. **数据库连接慢**: 未使用连接池或连接池配置不当
3. **首次加载开销**: Python 模块首次导入较慢

**优化建议**:
```python
# 1. 调整速率限制（针对测试环境）
@limiter.limit("100/minute")  # 提高限制

# 2. 配置数据库连接池

    DATABASE_URL,
    pool_size=10,
    max_overflow=20,
    pool_timeout=30
)

# 3. 启用应用预热
@app.on_event("startup")
async def startup():
    # 预加载常用数据
    pass
```
