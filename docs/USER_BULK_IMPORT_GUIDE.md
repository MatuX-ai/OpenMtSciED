# 用户批量导入功能使用指南

## 📋 功能概述

用户批量导入功能允许管理员通过CSV或Excel文件快速批量创建用户账户，支持字段映射、冲突检测和多种处理策略。

## 🚀 主要特性

- **多格式支持**: 支持CSV和Excel(.xlsx/.xls)文件格式
- **智能字段映射**: 自动识别多种字段名称变体
- **冲突检测**: 智能检测用户名和邮箱冲突
- **灵活处理策略**: 提供多种冲突处理选项
- **进度反馈**: 实时显示导入进度和结果
- **安全验证**: 完整的数据验证和权限控制

## 📁 支持的文件格式

### CSV格式
```
username,email,role
张三,zhangsan@example.com,user
李四,lisi@example.com,admin
王五,wangwu@example.com,user
```

### Excel格式
支持 `.xlsx` 和 `.xls` 格式的Excel文件，结构与CSV相同。

## 🔧 字段映射规则

### 必填字段
| 目标字段 | 可接受的源字段名 |
|---------|----------------|
| username | username, Username, 用户名 |
| email | email, Email, 邮箱 |

### 可选字段
| 目标字段 | 可接受的源字段名 |
|---------|----------------|
| role | role, Role, 角色 |

### 字段验证规则
- **用户名**: 3-50个字符，不能为空
- **邮箱**: 有效的邮箱格式，最大100字符
- **角色**: 必须是以下之一: user, admin, org_admin, premium

## ⚙️ 冲突处理策略

### 1. 跳过重复项 (skip) - 推荐
- 发现冲突时跳过该记录
- 继续导入其他有效记录
- 适合增量导入场景

### 2. 更新现有用户 (update)
- 更新已存在的用户信息
- 保持用户ID不变
- 适用于信息更新场景

### 3. 完全覆盖 (overwrite)
- 删除现有用户重新创建
- 用户ID会发生变化
- 谨慎使用

### 4. 发现冲突时停止 (error)
- 遇到第一个冲突就停止导入
- 返回详细的冲突信息
- 适合严格的验证场景

## 🔐 权限要求

- 仅**管理员**和**企业管理员**可以执行批量导入
- 普通用户无权访问此功能
- 所有操作都会记录审计日志

## 💻 使用方法

### React Admin界面使用

1. 登录管理员账户
2. 导航到"批量导入用户"页面
3. 点击上传区域或拖拽文件
4. 选择合适的冲突处理策略
5. 点击"开始导入"按钮
6. 等待导入完成查看结果

### Angular管理界面使用

1. 进入用户管理模块
2. 点击"批量导入"按钮
3. 选择CSV/Excel文件
4. 配置导入参数
5. 执行导入操作
6. 查看导入报告

### API调用方式

```bash
curl -X POST "http://localhost:8000/api/v1/auth/bulk-import" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -F "file=@users.csv" \
  -F "conflict_resolution=skip" \
  -F "generate_password=true"
```

## 🔍 常见问题解答

### Q: 导入的用户初始密码是什么？
A: 默认情况下会自动生成12位随机密码。如果禁用密码生成，则使用默认密码"123456"。

### Q: 如何处理导入失败的记录？
A: 系统会详细报告每个失败的原因，包括行号和具体错误信息。可以根据提示修正数据后重新导入。

### Q: 支持哪些字符编码？
A: 系统自动处理UTF-8编码，支持中文等Unicode字符。

### Q: 一次最多能导入多少用户？
A: 理论上没有硬性限制，但建议单次导入不超过1000条记录以保证性能。

### Q: 导入过程中出现错误怎么办？
A: 系统会回滚所有更改，确保数据一致性。可以修正文件后重新尝试导入。

## 🛠️ 故障排除

### 文件格式错误
- 确保文件扩展名正确(.csv/.xlsx/.xls)
- 检查文件是否损坏
- 验证编码格式为UTF-8

### 字段映射问题
- 确认列标题拼写正确
- 使用提供的模板文件作为参考
- 注意大小写敏感性

### 权限相关错误
- 确认使用管理员账户登录
- 检查API token是否有效
- 验证用户角色权限

### 性能问题
- 减少单次导入的数据量
- 关闭不必要的浏览器标签页
- 在网络状况良好的环境下操作

## 📝 最佳实践

1. **使用模板**: 始终使用系统提供的模板文件
2. **小批量测试**: 先用少量数据测试导入流程
3. **备份数据**: 导入前备份现有用户数据
4. **验证数据**: 导入前后都要验证数据完整性
5. **监控日志**: 关注系统日志中的警告和错误信息
