# 桌面端核心功能规划

## 1. 智能 STEM 搜索 (Intelligent STEM Search)
**目标：** 实现云端课程检索与全网 STEM 资源智能聚合。

- **混合搜索逻辑：** 
    - 优先检索本地 SQLite 数据库和云端 PostgreSQL 课程库。
    - 若结果不足，自动触发全网搜索（集成爬虫模块与搜索引擎 API）。
    - 利用 AI 对搜索结果进行摘要提取，生成标准化的学习目标和难度评级。
- **前端交互：** 
    - 在搜索栏提供“智能全网搜索”开关。
    - 展示搜索结果的来源可靠性标识（如：官方教材、开源社区、AI 推荐）。

## 2. 本地项目与课程构建 (Local Project & Course Builder)
**目标：** 支持用户在本地创建个性化课程，并通过下载云资源快速搭建。

- **我的项目/课程：** 
    - 用户可在本地创建“我的项目”，作为个性化课程的容器。
    - 支持从云端教程库、课件库一键下载并关联到本地项目。
- **资源管理：** 
    - 建立 `user_projects` 与 `project_resources` 的本地关联表。
    - 支持离线访问已下载的 STEM 教程和实验指导书。

## 3. 社区分享与积分体系 (Community Sharing & Points)
**目标：** 鼓励用户上传优质本地课程至云库，并通过积分排行榜激励贡献。

- **云端分享：** 
    - 用户可将本地构建的课程结构同步至云端 `user_resources` 表。
    - 其他用户下载该资源时，系统自动增加原作者的积分。
- **积分排行榜：** 
    - 在个人中心展示“贡献积分”及“社区贡献排行榜”。
    - 积分仅用于荣誉展示（排行榜），不涉及物质奖励，保持社区纯粹性。

## 4. 技术实现要点
- **后端 (FastAPI):** 扩展 `/api/v1/resources` 接口，增加 `smart-search`, `share`, `leaderboard` 端点。
- **桌面端 (Tauri + Rust):** 新增 `user_project.rs` 模块处理本地项目管理，并利用 `reqwest` 实现云端同步。
- **数据库:** 
    - 本地 SQLite 增加项目关联表。
    - 云端 PostgreSQL 增加 `UserResource` 和 `UserProfile` 模型以追踪贡献度。
