# 项目目录结构说明

## 根目录结构

```
OpenMTSciEd/
├── admin-web/                # 管理后台（Angular）
├── ai-service/               # AI服务（独立服务）
├── backend/                  # 后端核心服务（FastAPI）
├── data/                     # 数据文件
├── datasets/                 # 数据集和缓存
├── desktop-manager/          # Tauri + Angular 桌面客户端
├── docs/                     # 技术文档
├── frontend/                 # Web前端应用（Angular）
├── models/                   # AI模型文件
├── reports/                  # 报告输出
├── scripts/                  # 各种脚本工具
├── tests/                    # 测试文件
├── website/                  # 营销网站
├── .gitignore               # Git忽略配置
├── LICENSE                  # 许可证文件
├── NOTICE                   # 声明文件
├── README.md                # 项目说明
├── requirements.txt         # Python依赖
└── requirements.dev.txt     # 开发环境依赖
```

## 主要目录说明

### backend/
后端核心服务，基于 FastAPI 构建
- `openmtscied/` - 主应用包
  - `api/` - RESTful API 路由
  - `models/` - Pydantic 数据模型
  - `services/` - 业务逻辑服务
  - `data/` - 数据模型定义

### frontend/
Web 前端应用，基于 Angular 构建
- `src/app/` - 应用组件
  - `auth/` - 认证模块
  - `path-visualization/` - 路径可视化
  - `marketing-home/` - 营销首页

### admin-web/
管理后台，基于 Angular 构建

### desktop-manager/
桌面客户端，基于 Tauri + Angular 构建

### ai-service/
独立的 AI 服务项目

### data/
数据文件存储
- `databases/` - 数据库文件（SQLite）
- `course_library/` - 教程库
- `textbook_library/` - 课件库
- `hardware_projects.json` - 硬件项目数据
- `ai_learning_tasks.json` - AI 学习任务

### scripts/
各种脚本工具
- `db_management/` - 数据库管理脚本
- `data_processing/` - 数据处理脚本
- `scrapers/` - 资源爬取器
- `graph_db/` - Neo4j 图数据库相关脚本
- `parsers/` - 数据解析器
- `deployment/` - 部署脚本

### docs/
技术文档
- `01-项目概述/` - 项目介绍和变更日志
- `02-开发指南/` - 开发和贡献指南
- `03-核心技术/` - 核心技术文档
- `05-部署运维/` - 部署和运维指南

### datasets/
数据集和缓存文件
- `ai-edu-resources/` - AI 教育资源
- `processed/` - 处理后的数据
- `cache/` - 缓存文件

### models/
AI 模型文件
- `distilled/` - 蒸馏模型

### tests/
测试文件和测试脚本

### website/
营销网站静态文件

## 数据库文件

所有数据库文件统一存放在 `data/databases/` 目录中：
- `openmtscied.db` - 主数据库
- `ai_service.db` - AI 服务数据库
- `ai_edu.db` - AI 教育数据库
- `ai_edu_standalone.db` - AI 教育独立数据库
- `imato.db` - iMato 数据库

## 注意事项

1. 所有 `.db` 文件已在 `.gitignore` 中配置，不会被提交到 Git 仓库
2. 大型数据文件应使用 Git LFS 管理
3. 敏感信息应存储在 `.env` 文件中，不要提交到版本控制
