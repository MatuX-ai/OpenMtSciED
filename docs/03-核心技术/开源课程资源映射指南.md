# 开源课程资源映射指南

## 1. 映射目标与原则

OpenMTSciEd 的核心在于打通教程库（如 OpenSciEd）与课件库（如 OpenStax）。本指南定义了如何将这些异构资源整合进统一的 STEM 连贯学习路径引擎。

### 核心原则
*   **原子化 (Atomicity)**：将长篇教程拆解为最小知识单元（Knowledge Unit），确保能与教材章节精准对齐。
*   **双向索引 (Bidirectional Indexing)**：实现从现象驱动的实验到理论公式的双向检索。
*   **低成本适配 (Low-cost Adaptation)**：在映射过程中优先标记预算50元的硬件替代方案。

---

## 2. 元数据标准化规范 (Schema Definition)

为了实现跨库关联，所有导入的资源必须转换为以下统一结构：

### 2.1 统一标识符
*   **UUID**: 每个知识点或项目生成唯一的 esource_id。
*   **Source Tag**: 标记来源，例如 OS-Unit-001 (OpenSciEd) 或 OST-Phys-Ch3 (OpenStax)。

### 2.2 标签体系 (Tagging System)
| 维度 | 示例标签 | 作用 |
| :--- | :--- | :--- |
| **学科** | Physics, Chemistry, Biology, Engineering, Math | 确定知识所属领域 |
| **学段** | K-5, 6-8, 9-12, Undergraduate | 匹配用户认知水平 |
| **能力** | Observation, Hypothesis, Experiment, Modeling | 定义教学目标 |
| **硬件** | Arduino, ESP32, Sensor, No-Hardware | 筛选实验可行性 |

### 2.3 结构化字段
`json
{
  "title": "生态系统能量流动",
  "description": "探究生态系统中能量的输入、传递和散失过程。",
  "prerequisites": ["光合作用基础", "食物链概念"],
  "learning_outcomes": ["能绘制能量金字塔", "理解热力学第二定律在生物中的应用"]
}
`

## 3. 资源解析策略 (Parsing Strategy)

### 3.1 OpenSciEd (现象驱动教程)
*   **提取重点**：驱动性问题 (Driving Questions)、实验步骤、材料清单。
*   **处理方式**：利用 NLP 识别实验中的关键变量（如：光照、温度），将其转化为可搜索的关键词。

### 3.2 OpenStax (理论教材)
*   **提取重点**：章节目录、核心定义、数学公式、课后习题。
*   **处理方式**：建立公式-应用场景索引，例如将 =ma$ 关联到所有涉及力学的实验项目中。

### 3.3 格物斯坦 / stemcloud.cn (硬件项目)
*   **提取重点**：硬件 BOM 表、Blockly/Python 代码逻辑、电路原理图。
*   **处理方式**：解析硬件依赖，自动匹配对应的物理/电子学理论章节。

## 4. 关联算法逻辑 (Linking Logic)

系统通过以下三层逻辑建立资源间的关联：

1.  **关键词匹配 (Keyword Matching)**：基于 STEM 专业术语库进行初步筛选（如：匹配欧姆定律与电阻测量实验）。
2.  **语义相似度 (Semantic Similarity)**：利用向量模型计算教程描述与教材摘要的余弦相似度，发现隐性关联。
3.  **人工校验接口 (Human-in-the-loop)**：提供 JSON 修正入口，允许开发者手动调整 AI 推荐的关联权重。

## 5. 知识图谱存储示例 (Neo4j Cypher)

在 Neo4j 中，我们通过以下关系定义连贯路径：

`cypher
// 创建教程单元节点
CREATE (u:Unit {id: 'OS-001', title: '电路基础', source: 'OpenSciEd'})

// 创建教材章节节点
CREATE (c:Chapter {id: 'OST-PHY-12', title: '电流与电压', source: 'OpenStax'})

// 建立递进关系：教程需要该理论支撑
CREATE (u)-[:REQUIRES_THEORY]->(c)

// 建立硬件映射：教程使用了特定硬件
CREATE (h:Hardware {name: 'Arduino Uno'})
CREATE (u)-[:USES_COMPONENT]->(h)
`

---

## 6. 维护与更新

*   **版本控制**：每次开源课程更新（如 OpenStax 发布新版教材）时，需重新运行解析脚本并比对差异。
*   **质量监控**：定期检查关联准确率，确保没有错误的跨学段映射（如将大学量子力学直接关联到小学电路实验）。
