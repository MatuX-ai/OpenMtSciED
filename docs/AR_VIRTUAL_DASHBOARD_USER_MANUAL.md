# AR虚拟仪表盘用户手册

## 产品介绍

欢迎使用增强现实AR虚拟仪表盘！这是一款集成了专业级仪表显示、真实硬件传感器数据采集、高级数据分析功能的创新型测量工具。通过增强现实技术，您可以在真实环境中放置虚拟的专业测量仪器，并获得如同真实设备般的使用体验。

## 系统要求

### 硬件要求
- **Android设备**: Android 7.0及以上，支持ARCore
- **iOS设备**: iOS 11.0及以上，支持ARKit
- **处理器**: 四核1.5GHz以上
- **内存**: 4GB RAM以上推荐
- **摄像头**: 后置摄像头清晰，无遮挡
- **传感器**: 支持陀螺仪、加速度计

### 软件要求
- **操作系统**: Android 7.0+/iOS 11.0+
- **存储空间**: 至少500MB可用空间
- **网络连接**: WiFi或移动数据（用于在线功能）

### 可选硬件
- Arduino Uno/兼容板（用于真实传感器数据）
- 传感器模块（温度、湿度、光敏等）
- USB OTG线缆（Android设备）
- Lightning转USB相机适配器（iOS设备）

## 快速入门

### 1. 安装和启动
1. 从应用商店下载并安装应用
2. 首次启动时允许相机和存储权限
3. 选择"AR虚拟万用表"功能进入主界面

### 2. 基本操作流程
1. **环境准备**: 找一个光线充足、表面平整的区域
2. **放置仪器**: 相机对准平面，屏幕上会出现放置指示
3. **确认放置**: 点击屏幕确认放置虚拟万用表
4. **开始测量**: 点击万用表开始测量
5. **查看结果**: 实时数据显示在屏幕上方

## 详细功能说明

### 主要显示区域

#### 1. 专业仪表盘
- **模拟指针**: 仿真的机械指针显示当前数值
- **数字显示**: 高精度数字读数
- **状态指示**: 
  - 绿色"OK"：正常测量范围
  - 红色"OVER"：超量程警告
  - 黄色"LOW"：欠量程提示
- **单位显示**: 当前测量单位（V/A/Ω/Hz等）

#### 2. 历史数据图表
- **实时波形**: 显示最近测量数据的变化趋势
- **数据点**: 可点击查看具体数值
- **时间轴**: 显示数据采集时间

#### 3. 控制面板
- **电源按钮**: 开启/关闭测量
- **模式切换**: 电压/电流/电阻/频率测量模式
- **专业视图**: 切换到详细专业仪表盘
- **设置**: 进入配置菜单

### 手势操作指南

#### 基本手势
- **单击**: 选择/激活功能
- **双击**: 切换专业仪表盘显示
- **长按**: 打开快捷菜单
- **拖拽**: 移动物体位置（开发中）

#### 高级手势
- **捏合缩放**: 双指张开/合拢调整仪器大小
- **旋转**: 双指旋转调整仪器角度
- **滑动**: 左右滑动切换测量模式
- **多指操作**: 三指及以上手势（特殊功能）

### 测量模式详解

#### 电压测量 (V)
- **量程**: 0-20V DC/AC
- **精度**: ±0.1V
- **适用**: 电池电压、电路电压测量

#### 电流测量 (A)
- **量程**: 0-10A DC/AC
- **精度**: ±0.01A
- **适用**: 电路电流、设备功耗测量

#### 电阻测量 (Ω)
- **量程**: 0-1000Ω
- **精度**: ±1Ω
- **适用**: 元器件阻值、线路通断测量

#### 频率测量 (Hz)
- **量程**: 0-100Hz
- **精度**: ±0.1Hz
- **适用**: 信号频率、周期测量

#### 温度测量 (°C)
- **量程**: -40°C 到 +125°C
- **精度**: ±0.5°C
- **适用**: 环境温度、设备温度监测

## 硬件集成功能

### 设备连接设置

#### USB连接（Android）
1. 准备Arduino设备和传感器
2. 通过USB OTG线连接手机和Arduino
3. 点击界面中的蓝牙图标
4. 选择对应的USB设备
5. 等待连接成功提示

#### 蓝牙连接（iOS）
1. Arduino设备需支持蓝牙模块
2. 在设置中配对蓝牙设备
3. 应用中点击蓝牙连接按钮
4. 选择已配对的设备
5. 完成连接

### 传感器配置

#### 模拟传感器
- **连接方式**: Arduino模拟引脚A0-A5
- **典型应用**: 温度传感器、光敏电阻
- **配置**: 自动识别，无需手动设置

#### 数字传感器
- **连接方式**: Arduino数字引脚D2-D13
- **典型应用**: 按钮、继电器、数字传感器
- **配置**: 在设置中指定引脚功能

#### PWM输出
- **连接方式**: Arduino PWM引脚3,5,6,9,10,11
- **典型应用**: LED调光、电机调速
- **控制**: 通过界面滑块调节占空比

### 校准功能

#### 自动校准
1. 进入设置菜单
2. 选择"自动校准"
3. 按照提示进行零点和满量程校准
4. 保存校准参数

#### 手动校准
1. 进入设置菜单
2. 选择"手动校准"
3. 输入已知标准值
4. 调整斜率和偏移参数
5. 验证校准结果

## 故障排除

### 常见问题解决

#### AR功能问题
**问题**: 无法检测到平面
**解决**: 
- 确保环境光线充足
- 移动设备寻找合适平面
- 清洁摄像头镜头
- 重启应用

**问题**: 虚拟物体抖动严重
**解决**:
- 保持设备稳定
- 避免快速移动
- 检查是否有磁性干扰
- 调整AR设置中的稳定性参数

#### 硬件连接问题
**问题**: 无法连接USB设备
**解决**:
- 检查USB OTG线是否正常
- 确认Arduino供电正常
- 重新插拔USB连接
- 检查设备管理器权限

**问题**: 传感器数据异常
**解决**:
- 检查传感器接线是否正确
- 验证Arduino程序是否正确烧录
- 进行传感器校准
- 更换传感器测试

#### 性能问题
**问题**: 应用运行卡顿
**解决**:
- 关闭其他后台应用
- 降低显示质量和刷新率
- 清理应用缓存
- 重启设备

### 错误代码说明

| 错误代码 | 含义 | 解决方法 |
|---------|------|----------|
| ERR_001 | 相机权限被拒绝 | 在设置中开启相机权限 |
| ERR_002 | ARCore/ARKit不支持 | 更换支持AR的设备 |
| ERR_003 | 硬件连接失败 | 检查USB连接和设备驱动 |
| ERR_004 | 传感器数据错误 | 检查传感器连接和校准 |
| ERR_005 | 内存不足 | 关闭其他应用释放内存 |
