## 🔐 认证流程

### 登录流程
1. 用户在website点击"登录"
2. 跳转到 `http://localhost:4200/login`
3. 在desktop-manager完成登录
4. 登录后自动返回website（如果从website跳转）
5. localStorage中保存user信息和access_token

### 访问控制
两个页面都有登录检查：

```javascript
function checkAuth() {
    const user = localStorage.getItem('user');
    if (!user) {
        // 未登录，跳转到登录页
        window.location.href = 'http://localhost:4200/login';
        return false;
    }
    return true;
}
```

### Token使用
profile.html 使用 access_token 调用后端API：

```javascript
const token = localStorage.getItem('access_token');
fetch(`${API_BASE_URL}/me`, {
    headers: {
        'Authorization': `Bearer ${token}`
    }
});
```

## 📊 数据流

### Dashboard页面
```
用户访问 dashboard.html
    ↓
检查登录状态 (localStorage.user)
    ↓
未登录 → 跳转到 login
    ↓
已登录 → 加载模拟数据
    ↓
显示统计、活动、推荐
    ↓
点击"打开桌面端应用" → http://localhost:4200/dashboard
```

### Profile页面
```
用户访问 profile.html
    ↓
检查登录状态
    ↓
未登录 → 跳转到 login
    ↓
已登录 → 调用 GET /auth/me
    ↓
显示用户信息
    ↓
用户编辑资料 → PUT /auth/me/profile
    ↓
用户修改密码 → POST /auth/me/password
    ↓
显示成功/失败提示
```
